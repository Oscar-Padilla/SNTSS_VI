### Decisión Principal: Sitio Web vs Aplicación Móvil

- Se decidió desarrollar un sitio web en lugar de una aplicación móvil
- Razones principales: mayor facilidad de mantenimiento, mejor acceso para trabajadores de edad avanzada que no están familiarizados con tecnologías, no requiere instalación ni actualizaciones
- El sitio web será accesible desde cualquier dispositivo (computadora o navegador de celular)
- Comparte la misma base de datos para vista de escritorio y celular

### Sistema de Códigos QR para Control de Eventos

- Se implementarán códigos QR para mejor control y evitar gastos innecesarios de papel e tinta
- Cada trabajador tendrá un identificador único con código QR que puede reutilizarse para múltiples eventos
- El mismo QR puede servir para Día de la Madre, entrega de juguetes, becas, etc.
- El QR puede funcionar como firma electrónica para cumplir con requisitos de transparencia
- Sistema permitirá escanear códigos QR al llegar al evento para registro automático
- Se podrá generar listas de asistencia (ej: de 800 inscritos, 500 asistieron)
- Los códigos QR pueden agregarse a la billetera digital del celular

### Gestión de Documentos y Base de Datos

- El sistema no eliminará datos, sino que los filtrará por año para mantener historial completo
- Documentos en PDF para la mayoría de archivos, JPEG para credencialización
- Sistema podrá recordar documentos ya subidos previamente para no solicitarlos nuevamente
- Se solicitó que la plataforma unifique todos los procesos de las diferentes secretarías

### Necesidades por Secretaría

**Igualdad Sustantiva (Romy Casanova):**

- Maneja eventos como Día de las Madres, Día del Niño, Día del Padre
- Requiere: credencial, acta de nacimiento, afiliación, CURP para inscripciones de cursos de verano y eventos
- Actualmente usa Google Forms que se llena rápido y requiere pagos por almacenamiento adicional
- Problema actual: documentos a veces se suben en formato incorrecto (.png en lugar de .pdf)

**Previsión Social (Cecilia):**

- Maneja eventos contractuales con aproximadamente 3,600 personas
- Requiere verificación de último talón, credencial y contrato para generar invitaciones
- Necesita control de asistencia a eventos y validación de derechos

### Funcionalidades del Sitio Web

- Portal informativo con identidad de la sección: quiénes somos, contactos, redes sociales
- Sección de noticias y artículos (actualmente se maneja por Facebook)
- Sistema de filtros para estadísticas del personal por hospital o unidad
- Información de delegados sindicales, secretario general, secretario del interior
- Diseño amigable y visualmente atractivo, no como página gubernamental tradicional

### Administración y Acceso

- Cada secretaría podrá administrar únicamente su propia sección
- Los administradores podrán crear convocatorias, duración, requisitos, fotos, sin necesidad de Oscar
- La plataforma quedará lista para que las secretarías la alimenten en el futuro

### Cumplimiento Legal

- Se adherirá a la Ley de Protección de Datos Personales

### Proceso de Desarrollo Propuesto

1. Identificar necesidades y objetivos de cada secretaría 
2. Definir requisitos específicos (exportación a PDF/Excel, comparación de listas, etc.) 
3. Diseñar solución basada en estructura visual y contenido 
4. Crear casos de uso y diagramas de flujo de procesos  
5. Construir propuesta de costos 
6. Desarrollar prototipo visual
